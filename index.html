Introduction to easyEWAS

Yuting Wang, Meijie Jiang, Haocan Song, Xu Gao*

2024-04-01

Summary

easyEWAS is an R package designed for conducting Epigenome-Wide Association Study (EWAS) with Illumina Beadchip data for researchers less experienced in EWAS or parallel computing in population-based analyses. Users only need to provide sample phenotype data and DNA methylation values to easily perform EWAS analysis. The package supports three statistical methods, including linear model, linear mixed-effects model and Cox proportional-hazards model. Additionally, the use of parallel computing significantly improves computational efficiency. It also supports functions of the results visualization based on CMplot package, including Manhattan plots, QQ plots, etc.

Workflow

Installation

Users can install the easyEWAS package by running the following code:

devtools::install_github("ytwangZero/easyEWAS")

Example

Step 1 Initialize the EWAS module

Here, take the easyEWAS internal dataset as an example. After loading easyEWAS, the user needs to use the initEWAS() function to initialize an analysis module. This will generate a folder named result_easyEWAS under the working path specified by the user to store the EWAS analysis results. The function also returns an R6 object integrating all information, so the user needs to assign the result of the function to a new name, such as res. If default, the results will be stored in the current working path.

library(easyEWAS)
res <- initEwas(outpath = "default")
#> Complete initializing the EWAS module! Your EWAS results will be stored in D:/R/Rwork/oldversion/easyEWAS/vignettes/result_easyEWAS
#> 2024-03-11 01:27:08.398984
#> 0.09 sec elapsed

Step 3 Transform the variable types

This is an optional step where the user can transform certain variables in the sample dataset to a specified type. The transType function support both factor and numeric types

res <- transType(input = res, Vars = "cov1,cov2",TypeTo = "factor")

Step 4 Peform the EWAS analysis

This is the core step of the analysis. The user can select models based on the analysis requirements, including linear regression models, linear mixed effects models, and Cox proportional risk models. The function also supports parallel operations, and the number of physical cores required for operations can be specified in the num_cores parameter. The result will be automatically stored in a custom named.csv file. For the meaning of each parameter, refer to the manual of the R package.

res <- zeroEwas(input = res,
                file.name = "default",
                chipType = "EPICV2", # "EPICV2", "EPICV1", "450K"
                model = "lm", # "lm", "lmer", "cox"
                expo = "default",
                cov = NULL,
                random = NULL,
                time = NULL,
                status = NULL,
                adjustP = TRUE,
                num_cores = 3)
#> It will take some time, please be patient...

Step 5 Visualize the EWAS results

Visualize EWAS results based on the CMplot package, including Manhattan plots, QQ plots, etc. Optional parameters are the same as those in the CMplot function. When a users exposure variable is a categorical variable, the user can specify which class of p-values to use for results visualization.

res <- plotEwas(input = res,
                pval = "PVAL",
                file.name = "test")

Bootstrap for internal validation

Bootstrap is a statistical resampling technique used for internal validation and estimation of uncertainty in statistical models. Its particularly useful when the dataset is limited or when the underlying distribution of the data is not well known. Bootstrap can be a valuable tool for validating results from an EWAS. easyEWAS currently only provides the bootstrap function for linear regression models and linear mixed effects models where the independent variables are continuous variables. Users can run the following code to obtain the results of internal validation.

res <- bootEWAS(input = res,
                filterP = "PVAL", # FDR, Bonfferoni
                threshold = 0.05,
                CpGs = NULL,
                times = 100,
                seed = 123,
                file.name = "default")
#> It will take some time, please be patient...
#> Bootstrap analysis has been completed !
#> You can find results in D:/R/Rwork/oldversion/easyEWAS/vignettes/result_easyEWAS.
#> 2024-03-11 01:27:55.464417
#> 37.9 sec elapsed

Users can select sites to validate in two ways. One is to set the P-value column (PVAL, FDR, Bonfferoni) to be filtered and its filtering threshold (default is 0.05), and the other is to specify the name of the site to be verified directly in the CpGs parameter. The result is stored in res$bootres.
